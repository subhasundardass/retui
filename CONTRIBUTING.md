# Contributing to Retui

First of all, thank you for your interest in contributing to **Retui**! ❤️

Whether you're fixing a bug, improving documentation, suggesting a feature, or contributing code, your help is greatly appreciated.

## Ways to Contribute

You can contribute by:

- Reporting bugs
- Suggesting new features
- Improving documentation
- Fixing bugs
- Adding new widgets
- Improving performance
- Writing examples or tutorials
- Reviewing pull requests
- Helping answer questions from the community

Every contribution, big or small, helps make Retui better.

---

# Before You Start

Please:

- Read the README.
- Check existing Issues before creating a new one.
- Search existing Pull Requests to avoid duplicate work.
- Follow the project's Code of Conduct.

---

# Reporting Bugs

When reporting a bug, please include:

- Operating system
- Go version
- Retui version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or terminal recordings if applicable

A minimal reproducible example is always appreciated.

---

# Suggesting Features

Feature requests are welcome.

Please describe:

- The problem you're trying to solve
- Your proposed solution
- Possible alternatives
- Any additional context

Good feature requests make discussion much easier.

---

# Development Setup

Clone the repository:

```bash
git clone https://github.com/subhasundardass/retui.git
cd retui
```

Install dependencies:

```bash
go mod download
```

Run the project examples or tests as described in the README.

---

# Coding Guidelines

Please follow these guidelines:

- Write clean, readable Go code.
- Prefer simplicity over cleverness.
- Keep functions focused.
- Avoid unnecessary dependencies.
- Follow existing project structure and naming conventions.
- Add comments where they improve understanding.

---

# Commit Messages

Use clear commit messages.

Examples:

```
feat: add modal window support
fix: resolve keyboard focus issue
docs: improve README examples
refactor: simplify window rendering
test: add layout unit tests
```

---

# Pull Requests

Before submitting a Pull Request:

- Ensure the code builds successfully.
- Run all tests.
- Update documentation if necessary.
- Keep PRs focused on a single change.
- Provide a clear description of what changed and why.

Small, focused pull requests are easier to review and merge.

---

# Documentation

Documentation is just as valuable as code.

Improvements to:

- README
- Examples
- Tutorials
- API documentation

are always welcome.

---

# Questions

If you're unsure about anything, feel free to open a GitHub Discussion or Issue before starting work.

We're happy to discuss ideas and help contributors get started.

---

# Thank You

Thank you for taking the time to contribute to Retui.

Your support, feedback, and contributions help make the project better for everyone.

Happy coding! 🚀
